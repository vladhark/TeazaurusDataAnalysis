### 5.2. Аномалии (выбросы)

| Термин                                     | Определение                                                                                           | Связи                                                                             |
| ------------------------------------------ | ----------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------- |
| **[[Аномалия (выброс)]]**                  | Значение, значительно отклоняющееся от остальных наблюдений.                                          | Может быть ошибкой ввода, редким валидным случаем или систематическим искажением. |

#### 5.5. Токенизация текста

| Термин | Определение | Связи |
|--------|-------------|-------|
| **[[Токенизация текста (tokenization)]]** | Процесс разбиения текста на меньшие единицы (токены): слова, символы или субслова. | Используется для построения признаков из текста. |
| **[[Word‑level токенизация]]** | Разбиение текста на слова и знаки пунктуации (например, с помощью `WordPunctTokenizer` из NLTK). | Удобна для интерпретации, но приводит к большому размеру словаря. |
| **[[Char‑level токенизация]]** | Разбиение текста на отдельные символы (включая пробелы и знаки). | Маленький словарь, но очень длинные последовательности. |
| **[[Byte‑Pair Encoding (BPE)]]** | Алгоритм субсловной токенизации: начинается с символов, итеративно объединяет самые частые пары токенов до достижения нужного размера словаря. | Компромисс между word и char‑токенизацией; эффективно работает с редкими словами. |
| **[[WordPunctTokenizer]]** | Токенизатор из библиотеки NLTK, разделяющий строку на слова и пунктуацию. | Прост в использовании, подходит для английских текстов. |

#### 5.6. Векторное представление текста (эмбеддинги)

| Термин | Определение | Связи |
|--------|-------------|-------|
| **[[Эмбеддинг (embedding)]]** | Векторное представление токена (слова, субслова) в непрерывном пространстве, обученное так, чтобы семантически близкие слова имели близкие векторы. | Основа для [[Word2Vec]], GloVe, [[FastText]]. |
| **Word2Vec** | Алгоритм обучения эмбеддингов слов с помощью нейронной сети. Две архитектуры: CBOW (предсказание слова по контексту) и Skip-gram (предсказание контекста по слову). | Реализован в `gensim.models.Word2Vec`. |
| **[[GloVe (Global Vectors)]]** | Метод получения эмбеддингов на основе факторизации матрицы встречаемости слов (co‑occurrence matrix) в корпусе. | Модель `glove-twitter-100` в `gensim.downloader`. |
| **FastText** | Расширение Word2Vec, учитывающее n-граммы символов. Позволяет получать эмбеддинги для редких слов и слов, отсутствовавших в обучающем корпусе (out‑of‑vocabulary). | Модель `fasttext-wiki-news-subwords-300` в `gensim.downloader`. |
| **[[Предобученная модель эмбеддингов]]** | Готовая модель, обученная на большом корпусе (Wikipedia, Twitter, новости). Загружается через `gensim.downloader.api.load()`. | Используется без дополнительного обучения на собственном корпусе. |
| **[[Векторизация текста через суммирование (sum pooling)]]** | Получение вектора текста как суммы векторов всех входящих в него токенов (с использованием предобученных эмбеддингов). | Применяется для классификации текстов и поиска похожих предложений. |
| **[[Векторизация текста через среднее (mean pooling)]]** | Вектор текста = среднее арифметическое векторов всех токенов. | Альтернатива суммированию; меньше зависит от длины текста. |

#### 5.7. Bag of Words и TF‑IDF

| Термин | Определение | Связи |
|--------|-------------|-------|
| **[[Bag of Words (BoW)]]** | Представление текста в виде вектора, где каждый элемент соответствует частоте (количеству вхождений) определённого слова из предварительно построенного словаря. | Не учитывает порядок слов, прост в реализации. |
| **[[TF‑IDF (Term Frequency – Inverse Document Frequency)]]** | Метод взвешивания слов, учитывающий частоту слова в документе (TF) и редкость слова в корпусе (IDF). Формула: `tfidf(t,d) = tf(t,d) · log(N / df(t) + 1)`. | Улучшает BoW, подавляя частые, но малоинформативные слова. |

## 17. Модель Buy Till You Die (BTYD) / BG/NBD

| Термин | Определение |
|--------|-------------|
| **BG/NBD (Beta-Geometric / Negative Binomial Distribution)** | Вероятностная модель для прогнозирования количества будущих покупок и вероятности оттока клиентов на основе истории транзакций. |
| **«Смерть» клиента (churn)** | Клиент с вероятностью p перестаёт совершать покупки навсегда (геометрическое распределение). |
| **Интенсивность покупок λ** | Время до следующей покупки ~ Exp(λ); λ имеет гамма-распределение (гетерогенность). |
| **Параметры модели** | α, β — параметры гамма-распределения для λ; αₚ, βₚ — параметры бета-распределения для p. |
| **Recency (давность)** | Время от первой до последней покупки (в днях). |
| **Frequency (частота повторных покупок)** | Количество повторных покупок (total_purchases – 1). |
| **T (возраст клиента)** | Время от первой покупки до текущей даты (или конца периода наблюдения). |
| **Функция выживания S(t)** | Вероятность того, что клиент «жив» (не ушёл) спустя время t после последней покупки. |
| **Функция риска (hazard) h(t)** | Вероятность «смерти» в момент t при условии, что клиент дожил до t; h(t) = 1 – S(t) для дискретного времени. |
| **Прогноз числа покупок на следующие N дней** | Симуляция или аналитическая формула (conditional expectation) на основе оценённых параметров α, β, αₚ, βₚ, recency, frequency, T. |

## 24. Оценка качества моделей (метрики)

| Термин                                 | Определение                                                                                                                                         | Формула / Примечание                                                                                                          |
| -------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------- |
| **[[MAE (Mean Absolute Error)]]**      | Средняя абсолютная ошибка регрессии.                                                                                                                | `(1/n) Σ\|yᵢ – ŷᵢ\|                                                                                                           |
| **[[RMSE (Root Mean Squared Error)]]** | Корень из среднего квадрата ошибки; сильнее штрафует за большие отклонения.                                                                         | `√[(1/n) Σ(yᵢ – ŷᵢ)²]`.                                                                                                       |
| **[[R² (коэффициент детерминации)]]**  | Доля дисперсии, объяснённая моделью.                                                                                                                | `1 – Σ(yᵢ–ŷᵢ)² / Σ(yᵢ–ȳ)²`.                                                                                                   |

Бесполезна при сильном дисбалансе классов.                                                           |
| **[[Precision (точность)]]**           | Доля объектов, отнесённых к классу 1, которые действительно являются классом 1.                                                                     | `TP/(TP+FP)`.                                                                                                                 |
| **[[Recall (полнота)]]**               | Доля реальных объектов класса 1, правильно предсказанных.                                                                                           | `TP/(TP+FN)`.                                                                                                                 |
| **[[F1-score]]**                       | Среднее гармоническое precision и recall.                                                                                                           | `2·(Prec·Rec)/(Prec+Rec)`.                                                                                                    |
| **[[ROC-AUC (Area Under ROC Curve)]]** | Площадь под ROC-кривой (зависимость True Positive Rate от False Positive Rate). Значение 0,5 – случайное угадывание, 1,0 – идеальный классификатор. | `sklearn.metrics.roc_auc_score`, `roc_curve`. Широко используется для бинарной классификации, устойчива к дисбалансу классов. |
| **Silhouette score**                   | Оценка качества кластеризации. Высокое значение (близкое к 1) означает хорошо разделённые компактные кластеры.                                      | Для KMeans.                                                                                                                   |
| **Inertia**                            | Сумма квадратов расстояний до центроидов. Используется для выбора числа кластеров (метод локтя).                                                    |                                                                                                                               |

## 30. Дрейф данных (Data Drift)

| Термин                                       | Определение                                                                             | Связи                                                                                                                                          |
| -------------------------------------------- | --------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| **[[Data Drift]]**                           | Изменение статистических свойств данных между выборками (например, между train и test). | Может привести к ухудшению модели в эксплуатации.                                                                                              |
| **[[Covariate Drift]]**                      | Изменение распределения признаков (X) при неизменном законе.                            | Обнаруживается сравнением `X_train` и `X_test`.                                                                                                |
| **[[Label Drift]]**                          | Изменение распределения целевой переменной (y).                                         | Характерно для задач с меняющейся частотой классов.                                                                                            |
| **[[Concept Drift]]**                        | Изменение связи между признаками и целевой переменной.                                  | Сложнее всего обнаружить, требует мониторинга во времени.                                                                                      |
| **[[PSI (Population Stability Index)]]**     | Метрика изменения распределения числового признака между двумя выборками.               | `PSI = Σ (Actual% – Expected%) * ln(Actual%/Expected%)`. Значения: <0.1 – стабильно, 0.1–0.2 – небольшие изменения, ≥0.2 – значительный дрейф. |
| **[[CSI (Characteristic Stability Index)]]** | PSI для категориальных признаков (аналогичная формула по категориям).                   | Позволяет оценить дрейф категориальных переменных.                                                                                             |

## 31. Статистические тесты для обнаружения дрейфа

| Термин | Определение | Применение |
|--------|-------------|-------------|
| **[[KS-тест (Kolmogorov–Smirnov)]]** | Непараметрический тест равенства распределений двух непрерывных выборок. | `scipy.stats.ks_2samp()`. H₀: распределения одинаковы (дрейфа нет). p-value < 0.05 – дрейф значим. |
| **[[t-тест Стьюдента]]** | Проверка равенства средних двух выборок (предполагается нормальность или большой объём). | `scipy.stats.ttest_ind()`. H₀: средние равны. |

## 39. PCA (Principal Component Analysis)

| Термин                                             | Определение                                                                                                                                 | Связи                                                                                 |
| -------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------- |
| **PCA**                                            | Линейный метод понижения размерности, находящий ортогональные направления (главные компоненты), вдоль которых дисперсия данных максимальна. | `sklearn.decomposition.PCA`.                                                          |
| **[[Главная компонента (principal component)]]**   | Нормализованный собственный вектор ковариационной матрицы, соответствующий наибольшему собственному значению.                               | Компоненты упорядочены по убыванию объяснённой дисперсии.                             |
| **[[Объяснённая дисперсия (explained variance)]]** | Доля дисперсии исходных данных, сохраняемая при проекции на данную главную компоненту.                                                      | Доступна в `pca.explained_variance_ratio_`.                                           |
| **[[Суммарная объяснённая дисперсия]]**            | Сумма долей дисперсии для первых $k$ компонент. Показывает, сколько информации потеряно при сжатии до $k$ измерений.                        | Используется для выбора оптимальной размерности (правило локтя).                      |
| **Правило локтя (elbow rule)**                     | Выбор количества компонент, после которого прирост суммарной объяснённой дисперсии становится незначительным.                               | Визуализируется графиком `explained_variance_ratio_` в порядке убывания.              |
| **Матрица компонент `pca.components_`**            | Матрица размера `(n_components, n_features)`, строки которой являются главными компонентами (направлениями в исходном пространстве).        | Каждая компонента может быть визуализирована как «изображение» (например, для MNIST). |
| **[[Стандартизация для PCA]]**                     | Данные должны быть центрированы (вычтено среднее) и масштабированы (приведены к единичной дисперсии).                                       | Влияет на результат, поскольку PCA чувствителен к масштабу признаков.                 |

## 40. t-SNE (t-distributed Stochastic Neighbor Embedding)

| Термин                                            | Определение                                                                                                                                                | Связи                                                                                                     |
| ------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------- |
| **t-SNE**                                         | Нелинейный метод понижения размерности, основанный на вероятностном сходстве точек и использующий распределение Стьюдента для низкоразмерного отображения. | `sklearn.manifold.TSNE` (также `openTSNE`, для ML поддерживает только размерность 2).                     |
| **Перплексия (perplexity)**                       | Эффективное число ближайших соседей.                                                                                                                       | Типичные значения: 5–50. Меньшие значения подчёркивают локальную структуру, большие – глобальную.         |
| **Скорость обучения (learning rate)**             | Шаг градиентного спуска при оптимизации расхождения Кульбака–Лейблера.                                                                                     | Типичные значения: 10–1000. При слишком малой – медленная сходимость, при слишком большой – расходимость. |
| **Инициализация (init)**                          | Начальное расположение точек в низкоразмерном пространстве. `'random'` – случайное, `'pca'` – с помощью PCA (часто даёт лучшие результаты).                | Параметр `init` в `TSNE`.                                                                                 |
| **Расхождение Кульбака–Лейблера (KL divergence)** | Мера различия между распределением вероятностей соседства в исходном и низкоразмерном пространстве. t-SNE минимизирует KL($P\|Q$).                         | $KL(P\|Q) = \sum_{i\neq j} p_{ij} \log\frac{p_{ij}}{q_{ij}}$                                              |
| **[[Интерпретация t-SNE]]**                       | Расстояния между кластерами не имеют количественного смысла, важна только относительная близость точек внутри кластеров.                                   | t-SNE не сохраняет глобальные расстояния и масштабы.                                                      |

## 41. UMAP (Uniform Manifold Approximation and Projection)

| Термин | Определение | Связи |
|--------|-------------|-------|
| **UMAP** | Нелинейный метод понижения размерности, улучшение t-SNE. Быстрее t-SNE и лучше сохраняет глобальную структуру. Зачастую имеет схожие с t-SNE кластеры, из-за тенденции сжимать кластеры, по возможности, стоит использовать вместе с t-SNE. | Библиотека `umap-learn`. |
| **n_neighbors** | Аналог перплексии; размер локального окружения. Меньшие значения фокусируются на локальной структуре, большие – на глобальной. | Параметр `n_neighbors` в `umap.UMAP`. |
| **min_dist** | Минимальное допустимое расстояние между точками в низкоразмерном пространстве. Меньшие значения приводят к более плотным кластерам. | Параметр `min_dist` (по умолчанию 0.1). |
| **Метрика (metric)** | Функция расстояния, используемая для построения графа соседства. По умолчанию `'euclidean'`. | Параметр `metric`. |

## 42. Сравнение методов понижения размерности

| Характеристика | PCA | t-SNE | UMAP |
|----------------|-----|-------|------|
| **Тип** | Линейный | Нелинейный | Нелинейный |
| **Сохраняет глобальную структуру** | Да (при достаточном числе компонент) | Нет | Частично |
| **Сохраняет локальную структуру** | Нет | Да | Да |
| **Скорость работы** | Высокая (на больших данных) | Низкая (O(n²) или O(n log n) с приближениями) | Высокая (быстрее t-SNE) |
| **Рекомендуемое применение** | Сжатие признаков для ML, предобработка перед t-SNE/UMAP | Визуализация, изучение кластеров, сжатие признаков для ML (только с `openTSNE`) | Визуализация, изучение кластеров (по возможности, применять и t-SNE), сжатие признаков для ML |
| **Чувствительность к параметрам** | Низкая | Высокая (perplexity, learning rate) | Средняя (n_neighbors, min_dist) |

| **Scattermap (карта-рассеяние)** | Слой точечных данных, наложенный на карту (обычно OpenStreetMap или стиль Mapbox). | `go.Scattermap(lat=..., lon=..., mode='markers', marker=..., text=...)`. |
| **Субплоты (subplots)** | Группа из нескольких графиков, расположенных в сетке и связанных общими данными/фильтрами. | `fig = make_subplots(rows, cols, subplot_titles)`, затем `fig.add_trace(trace, row, col)`. |
| **Настраиваемое всплывающее окно (customdata + hovertemplate)** | Механизм передачи дополнительных данных для отображения при наведении курсора на точку. | `customdata` – массив данных; `hovertemplate` – строка с плейсхолдерами `%{customdata[...]}`. |
| **Фильтрация через виджеты (updatemenus / buttons)** | Элементы управления, позволяющие динамически показывать/скрывать слои на графике. | `updatemenus` с `buttons`; `visible` массива трасс управляется через `args`. |
| **Связанные графики (cross‑filtering)** | Выбор категории в одном графике автоматически отображает соответствующие подмножества точек на других графиках. | Реализуется через `updatemenus` и изменение `visible` всех трасс. |
| **Экспорт в HTML** | Сохранение интерактивного графика в отдельный HTML-файл для просмотра в браузере. | `fig.write_html('filename.html')`. |

## 52. Преобразования данных для геоаналитики

| Термин | Определение | Связи |
|--------|-------------|-------|
| **[[Геодезическое расстояние (geodesic distance)]]** | Расстояние между двумя точками на поверхности земного эллипсоида (в отличие от евклидова). | `geopy.distance.geodesic(pointA, pointB, ellipsoid='GRS-80').miles` (или километры). |
| **Вычисление `distance_between_points`** | Новый признак – прямое расстояние между двумя точками (например, между координатами посадки и высадки). | Используется для сравнения с фактическим пройденным расстоянием (например, из одометра). |
| **Аномалии расстояния** | Случаи, когда фактическое расстояние значительно превышает геодезическое – возможный выброс или ошибка данных. | Отфильтровываются или анализируются отдельно. |
| **[[Час события (pickup_hour)]]** | Целочисленный признак (0–23), извлечённый из временной метки. | Позволяет анализировать суточные паттерны. |
| **[[Стоимость мили (cost_per_unit)]]** | Отношение стоимости услуги к пройденному расстоянию (например, `fare_amount / trip_distance`). | Характеризует тарификацию, может выявлять аномалии. |
| **[[Фильтрация по ограничивающему прямоугольнику (bounding box)]]** | Ограничение координат диапазонами широт и долгот, соответствующими исследуемой территории. | Удаляет выбросы (некорректные GPS-координаты). |
