## pandas

**Определение**: Библиотека Python для работы с табличными данными.
**Связи**: Используется вместе с numpy, seaborn.

## pd.Series

**Определение**: Одномерный массив с именованными индексами. Состоит из values и index.

## pd.DataFrame

**Определение**: Двумерная таблица из нескольких Series с общим индексом. Основной объект pandas.

## Индекс (index)

**Определение**: Метки строк в Series или DataFrame. Позволяет обращаться к строкам по имени.

## Метод .head(n)

**Определение**: Возвращает первые n строк DataFrame. Для первичного ознакомления.

## Метод .info()

**Определение**: Выводит типы данных и количество непустых значений. Помогает найти пропуски.

## Метод .describe()

**Определение**: Возвращает статистику по числовым столбцам. Даёт представление о распределении.

## Атрибут .shape

**Определение**: Кортеж (число строк, число столбцов). Быстрая оценка объёма данных.

## Атрибут .columns

**Определение**: Список названий столбцов. Используется для доступа по именам.

## NaN (Not a Number)

**Определение**: Значение, обозначающее пропуск или отсутствие данных. Влияет на вычисления (многие функции игнорируют NaN).

## Тип object

**Определение**: В pandas обозначает строковый тип или смешанные данные. Часто встречается в категориальных признаках.

## Тип category

**Определение**: Оптимизированный тип для категориальных данных с фиксированным набором значений. Экономят память.

## Метод .isna() / .isnull()

**Определение**: Возвращает булеву маску, где True — пропуск. Используется для подсчёта пропусков: df.isna().sum().

## Метод .fillna(value)

**Определение**: Заменяет пропуски указанным значением (константой, строкой, результатом вычислений). Можно заполнять медианой, модой, средним или значением из другого столбца.

## Заполнение медианой

**Определение**: Стратегия заполнения пропусков медианным значением по группе (например, по стране). Устойчиво к выбросам, используется для цен.

## Заполнение модой

**Определение**: Заполнение наиболее частым значением в столбце (или по группе). Для категориальных признаков.

## Метод .dropna()

**Определение**: Удаляет строки (или столбцы), содержащие пропуски. Применяется, когда пропусков мало или они не критичны.

## Уникальные значения (unique)

**Определение**: Все различные значения в столбце. Связанные методы: .unique(), .nunique().

## Частотное распределение (value_counts)

**Определение**: Таблица, показывающая, как часто встречается каждое значение. .value_counts(normalize=True) для долей.

## Дубликаты строк

**Определение**: Полностью совпадающие строки (по всем столбцам). Искажают статистику, завышают частоты.

## Метод .duplicated()

**Определение**: Возвращает булеву маску, указывающую на повторяющиеся строки (кроме первого вхождения). df.duplicated().sum() – количество дубликатов.

## Метод .drop_duplicates()

**Определение**: Удаляет дубликаты. По умолчанию оставляет первую запись. Можно настроить параметры keep и subset.

## Аномалия (выброс)

**Определение**: Значение, значительно отклоняющееся от остальных наблюдений. Может быть ошибкой ввода, редким валидным случаем или систематическим искажением.

## Бизнес-логика (для выбросов)

**Определение**: Определение границ аномалий на основе экспертных знаний (например, цена не может быть отрицательной). Наиболее интерпретируемый подход.

## IQR (межквартильный размах)

**Определение**: Разность между третьим и первым квартилями: IQR = Q3 – Q1. Используется для выявления выбросов: границы [Q1 – 1,5×IQR, Q3 + 1,5×IQR].

## Z-score (стандартизованная оценка)

**Определение**: z = (x – μ) / σ. Значения с |z| > 3 считаются выбросами. Требует приблизительно нормального распределения.

## Метод .quantile(q)

**Определение**: Возвращает значение q-го квантиля (q от 0 до 1). Используется для вычисления IQR.

## Метод .clip(lower, upper)

**Определение**: Ограничивает значения указанными нижней и верхней границами. Стратегия обработки выбросов без удаления.

## Сегментация

**Определение**: Разделение данных на категории на основе порогов (например, дешёвые / нормальные / дорогие). Альтернатива удалению выбросов.

## Визуализация выбросов

**Определение**: Гистограмма (.hist()) и ящик с усами (.boxplot()) помогают обнаружить аномалии. Быстрый способ увидеть хвосты распределения.

## Несогласованный текст

**Определение**: Проблемы: разные регистры, лишние пробелы, опечатки, разные форматы одной сущности. Мешает группировке и анализу.

## Метод .str.lower() / .str.upper()

**Определение**: Приводит строки к нижнему/верхнему регистру. Унификация регистра.

## Метод .str.strip()

**Определение**: Удаляет пробелы в начале и конце строки. Устраняет лишние пробелы.

## Метод .str.translate()

**Определение**: Удаляет или заменяет символы по таблице перевода. Используется с str.maketrans('', '', punctuation) для удаления знаков препинания.

## Метод .str.replace(pattern, repl)

**Определение**: Заменяет подстроки по шаблону (можно словарём). Для исправления опечаток и синонимов.

## Метод .str.contains(pattern)

**Определение**: Проверяет, содержит ли строка заданную подстроку (регулярное выражение). Полезна для анализа текстовых описаний (например, поиск слов «fruit»).

## Метод .str.split(delimiter)

**Определение**: Разделяет строку на части по разделителю, возвращает Series списков. Полезен, когда в одной ячейке перечислено несколько значений (например, жанры, страны).

## Метод .explode()

**Определение**: Преобразует каждый элемент списка в отдельную строку, дублируя остальные столбцы. Позволяет провести анализ частот по каждому элементу списка.

## Токенизация текста (tokenization)

**Определение**: Процесс разбиения текста на меньшие единицы (токены): слова, символы или субслова. Используется для построения признаков из текста.

## Word‑level токенизация

**Определение**: Разбиение текста на слова и знаки пунктуации (например, с помощью WordPunctTokenizer из NLTK). Удобна для интерпретации, но приводит к большому размеру словаря.

## Char‑level токенизация

**Определение**: Разбиение текста на отдельные символы (включая пробелы и знаки). Маленький словарь, но очень длинные последовательности.

## Byte‑Pair Encoding (BPE)

**Определение**: Алгоритм субсловной токенизации: начинается с символов, итеративно объединяет самые частые пары токенов до достижения нужного размера словаря. Компромисс между word и char‑токенизацией; эффективно работает с редкими словами.

## WordPunctTokenizer

**Определение**: Токенизатор из библиотеки NLTK, разделяющий строку на слова и пунктуацию. Прост в использовании, подходит для английских текстов.

## Эмбеддинг (embedding)

**Определение**: Векторное представление токена (слова, субслова) в непрерывном пространстве, обученное так, чтобы семантически близкие слова имели близкие векторы. Основа для Word2Vec, GloVe, FastText.

## Word2Vec

**Определение**: Алгоритм обучения эмбеддингов слов с помощью нейронной сети. Две архитектуры: CBOW (предсказание слова по контексту) и Skip-gram (предсказание контекста по слову). Реализован в gensim.models.Word2Vec.

## GloVe (Global Vectors)

**Определение**: Метод получения эмбеддингов на основе факторизации матрицы встречаемости слов (co‑occurrence matrix) в корпусе. Модель glove-twitter-100 в gensim.downloader.

## FastText

**Определение**: Расширение Word2Vec, учитывающее n-граммы символов. Позволяет получать эмбеддинги для редких слов и слов, отсутствовавших в обучающем корпусе (out‑of‑vocabulary). Модель fasttext-wiki-news-subwords-300 в gensim.downloader.

## Предобученная модель эмбеддингов

**Определение**: Готовая модель, обученная на большом корпусе (Wikipedia, Twitter, новости). Загружается через gensim.downloader.api.load(). Используется без дополнительного обучения на собственном корпусе.

## Векторизация текста через суммирование (sum pooling)

**Определение**: Получение вектора текста как суммы векторов всех входящих в него токенов (с использованием предобученных эмбеддингов). Применяется для классификации текстов и поиска похожих предложений.

## Векторизация текста через среднее (mean pooling)

**Определение**: Вектор текста = среднее арифметическое векторов всех токенов. Альтернатива суммированию; меньше зависит от длины текста.

## Bag of Words (BoW)

**Определение**: Представление текста в виде вектора, где каждый элемент соответствует частоте (количеству вхождений) определённого слова из предварительно построенного словаря. Не учитывает порядок слов, прост в реализации.

## TF‑IDF (Term Frequency – Inverse Document Frequency)

**Определение**: Метод взвешивания слов, учитывающий частоту слова в документе (TF) и редкость слова в корпусе (IDF). Формула: tfidf(t,d) = tf(t,d) · log(N / df(t) + 1). Улучшает BoW, подавляя частые, но малоинформативные слова.

## Логическая индексация

**Определение**: Выбор строк по условию (булевой маске). df[условие].

## Метод .loc[]

**Определение**: Доступ по меткам или булевым условиям. df.loc[условие, столбцы].

## Комбинация условий

**Определение**: Использование & (И) и | (ИЛИ) с обязательными скобками. (условие1) & (условие2).

## Прямое присваивание (нового признака)

**Определение**: df['новый'] = значения. Изменяет исходный DataFrame.

## Функция np.where()

**Определение**: Условное создание значений: np.where(условие, true_val, false_val). Бинарная или вложенная классификация.

## Функция pd.cut()

**Определение**: Разбивает непрерывную переменную на интервалы (корзины). pd.cut(x, bins, labels).

## Лямбда-функции (lambda)

**Определение**: Безымянная функция для простых преобразований. Используется с .apply() или в качестве аргумента в других методах.

## Производный признак

**Определение**: Новый столбец, полученный из существующих (например, сумма, разность, категория на основе квантилей). Улучшает качество анализа.

## One‑hot encoding на основе токенов

**Определение**: Для отобранных токенов создаётся бинарный признак: 1 – токен присутствует в тексте, 0 – отсутствует. Применяется после токенизации; результат – таблица признаков для классификации.

## Разница частот токенов между классами

**Определение**: Абсолютная разность долей встречаемости токена в позитивных и негативных текстах. Позволяет выбрать наиболее информативные токены для one‑hot encoding.

## Группировка (groupby)

**Определение**: Разбиение данных на группы по значениям одного или нескольких столбцов.

## Агрегирующая функция

**Определение**: Функция, вычисляющая сводную статистику по каждой группе (среднее, сумма, медиана и т.д.).

## Метод .agg()

**Определение**: Применение нескольких агрегирующих функций одновременно.

## Корреляция

**Определение**: Статистическая мера связи между двумя числовыми признаками. Значение от -1 до 1.

## Коэффициент корреляции Пирсона (Pearson)

**Определение**: Мера линейной зависимости. Чувствителен к выбросам. Требует приблизительно нормального распределения.

## Коэффициент ранговой корреляции Спирмена (Spearman)

**Определение**: Мера монотонной (необязательно линейной) зависимости. Основан на рангах. Более устойчив к выбросам.

## Метод .corr(method=...)

**Определение**: Вычисляет корреляционную матрицу для числовых столбцов DataFrame. method = 'pearson' (по умолчанию) или 'spearman'.

## Косинусное сходство (cosine similarity)

**Определение**: Мера схожести двух векторов, определяемая как косинус угла между ними. Формула: cos(θ) = (A·B) / (||A||·||B||). Используется для поиска ближайших векторов, в том числе для поиска похожих предложений (sklearn.metrics.pairwise.cosine_similarity).

## Гистограмма

**Определение**: График распределения частот числового признака. .plot.hist(), .hist().

## Ящик с усами (boxplot)

**Определение**: График, показывающий медиану, квартили и возможные выбросы. .plot.box(). Используется для обнаружения аномалий.

## Count plot (столбчатая диаграмма частот)

**Определение**: График, показывающий количество (или долю) наблюдений для каждой категории. sns.countplot() или df['column'].value_counts().plot(kind='bar').

## Тепловая карта корреляций

**Определение**: Визуализация корреляционной матрицы с помощью цветовой шкалы. sns.heatmap(df.corr(), annot=True, cmap='coolwarm').

## numpy.random

**Определение**: Модуль для генерации псевдослучайных чисел из различных распределений. np.random.geometric(p, size), np.random.exponential(scale, size).

## scipy.stats

**Определение**: Модуль, предоставляющий объекты распределений с методами .rvs(), .pdf(), .cdf(), .ppf(), .pmf(). Более медленный, но удобный для теоретических расчётов.

## Фиксация seed

**Определение**: np.random.seed(42) — обеспечивает воспроизводимость результатов. Обязательно при моделировании.

## gaussian_kde

**Определение**: Класс в scipy.stats для построения сглаженной оценки плотности по выборке. kde = gaussian_kde(data, bw_method='scott').

## Параметр bw_method (bandwidth)

**Определение**: Регулирует «гладкость» оценки: малые значения → высокая детализация (шум), большие → излишнее сглаживание. bw_method='scott' (оптимально по умолчанию), bw_method=0.5 (для малых выборок).

## Аннотация со стрелкой (в matplotlib)

**Определение**: ax.annotate(text, xy, xytext, arrowprops) — выделение точек с визуальной связью. Указание на «тяжёлые хвосты» или бимодальность.

## Текстовый блок с фоном (matplotlib)

**Определение**: ax.text(x, y, text, bbox={'facecolor':'yellow', 'alpha':0.5, 'pad':5}). Пояснение параметров смеси, интерпретация графиков.

## Вертикальные / горизонтальные линии (matplotlib)

**Определение**: ax.axvline(x, color, linestyle), ax.axhline(y). Отметка средних значений, медианы, порогов.

## Заливка области (fill_between)

**Определение**: ax.fill_between(x, y1, y2, color, alpha). Визуализация доверительных интервалов, области между кривыми (например, функция риска).

## Заливка интервала (axvspan)

**Определение**: ax.axvspan(xmin, xmax, color, alpha). Выделение доверительного интервала прогноза.

## Сетка графиков (subplots)

**Определение**: fig, axes = plt.subplots(nrows, ncols, figsize, sharex, sharey). Сравнение нескольких распределений или эволюции процесса (например, приближение геометрического распределения к экспоненциальному).

## Параметры гистограммы (density=True)

**Определение**: density=True — нормализация (площадь = 1); bins=range(min, max+1) — для дискретных данных. Сравнение эмпирической гистограммы с теоретической плотностью.

## Легенда (legend)

**Определение**: ax.legend() — отображает подписи элементов. Обязательна при нескольких кривых на одном графике.

## Тип datetime (в pandas)

**Определение**: pd.to_datetime() — преобразование строки в дату.

## Разница между датами (pandas)

**Определение**: (date1 - date2).days — число дней между событиями.

## Группировка по времени (pandas)

**Определение**: df.groupby(pd.Grouper(key='date', freq='M')) — агрегация по месяцам.

## Геометрическое распределение Geom(p)

**Определение**: Распределение числа испытаний Бернулли до первого успеха (включительно). scipy.stats.geom. Используется в BTYD для моделирования числа покупок до «смерти» пользователя.

## Экспоненциальное распределение Exp(λ)

**Определение**: Распределение времени между независимыми событиями в пуассоновском потоке. scipy.stats.expon(scale=1/λ). В BTYD — время до следующей покупки.

## Бета-распределение Beta(α, β)

**Определение**: Распределение вероятности на отрезке [0,1]; гибко моделирует гетерогенность параметра p. scipy.stats.beta. Используется в BG/NBD для вариабельности вероятности «смерти».

## Гамма-распределение Gamma(α, β)

**Определение**: Распределение положительной случайной величины; моделирует гетерогенность интенсивности покупок λ. scipy.stats.gamma(a=α, scale=1/β).

## Смесь распределений (mixture distribution)

**Определение**: Линейная комбинация нескольких распределений с весами (вероятностями смеси). Пример: загрязнённое нормальное распределение: (1-ε)·N(μ,σ²) + ε·N(μ,ασ²).

## BG/NBD (Beta-Geometric / Negative Binomial Distribution)

**Определение**: Вероятностная модель для прогнозирования количества будущих покупок и вероятности оттока клиентов на основе истории транзакций.

## «Смерть» клиента (churn) в BTYD

**Определение**: Клиент с вероятностью p перестаёт совершать покупки навсегда (геометрическое распределение).

## Интенсивность покупок λ (в BTYD)

**Определение**: Время до следующей покупки ~ Exp(λ); λ имеет гамма-распределение (гетерогенность).

## Параметры BG/NBD (α, β, αₚ, βₚ)

**Определение**: α, β — параметры гамма-распределения для λ; αₚ, βₚ — параметры бета-распределения для p.

## Recency (давность) в BTYD

**Определение**: Время от первой до последней покупки (в днях).

## Frequency (частота повторных покупок) в BTYD

**Определение**: Количество повторных покупок (total_purchases – 1).

## T (возраст клиента) в BTYD

**Определение**: Время от первой покупки до текущей даты (или конца периода наблюдения).

## Функция выживания S(t) (BTYD)

**Определение**: Вероятность того, что клиент «жив» (не ушёл) спустя время t после последней покупки.

## Функция риска (hazard) h(t) (BTYD)

**Определение**: Вероятность «смерти» в момент t при условии, что клиент дожил до t; h(t) = 1 – S(t) для дискретного времени.

## Прогноз числа покупок на следующие N дней (BTYD)

**Определение**: Симуляция или аналитическая формула (conditional expectation) на основе оценённых параметров α, β, αₚ, βₚ, recency, frequency, T.

## pymc_marketing.clv.BetaGeoModel

**Определение**: Класс для байесовской оценки параметров BG/NBD.

## Подготовка данных для BG/NBD

**Определение**: Для каждого клиента: frequency, recency, T.

## Метод .fit() (для BG/NBD)

**Определение**: Обучает модель (MCMC или вариационный вывод).

## Прогноз с помощью BetaGeoModel

**Определение**: .expected_purchases(future_t) — ожидаемое количество покупок за future_t дней.

## Регрессия (regression)

**Определение**: Задача предсказания непрерывной числовой переменной (например, цены алмаза). Используются модели: SGDRegressor, RandomForestRegressor.

## Классификация (classification)

**Определение**: Задача отнесения объекта к одному из дискретных классов (например, ушёл клиент или остался). Используются модели: LogisticRegression, RandomForestClassifier, CatBoostClassifier.

## Кластеризация (clustering)

**Определение**: Задача обучения без учителя: поиск скрытых групп в данных (например, сегментация покупателей). Основной алгоритм: KMeans.

## SGDRegressor

**Определение**: Линейная регрессия, обучаемая стохастическим градиентным спуском. Чувствительна к масштабу признаков.

## RandomForestRegressor

**Определение**: Ансамбль решающих деревьев, не требует нормализации, лучше справляется с нелинейностями.

## LogisticRegression

**Определение**: Линейный классификатор с сигмоидной функцией активации. Используется для бинарной классификации.

## RandomForestClassifier

**Определение**: Ансамбль решающих деревьев для классификации. Устойчив к выбросам и не требует масштабирования.

## CatBoostClassifier

**Определение**: Градиентный бустинг на деревьях, эффективно работает с категориальными признаками без предварительного кодирования.

## KMeans

**Определение**: Алгоритм кластеризации, минимизирующий сумму квадратов расстояний от точек до центров кластеров (инерцию).

## Inertia

**Определение**: Сумма квадратов расстояний от каждого наблюдения до центроида его кластера. Убывает с ростом числа кластеров.

## Silhouette score

**Определение**: Метрика качества кластеризации, оценивающая, насколько точка похожа на свой кластер по сравнению с соседними. Принимает значения от -1 до 1.

## OneHotEncoder

**Определение**: Преобразование категориального признака в набор бинарных столбцов (по одному на каждую категорию). Не создаёт ложного порядка. Увеличивает размерность, но безопасен для любых моделей.

## OrdinalEncoder / LabelEncoder

**Определение**: Замена категорий целыми числами (0, 1, 2…). Вносит искусственный порядок. Опасен для линейных моделей, но допустим для деревьев.

## StandardScaler

**Определение**: Нормализация: вычитает среднее и делит на стандартное отклонение (получает распределение с μ=0, σ=1). Обязателен для SGD, полезен для логистической регрессии; не влияет на деревья.

## ColumnTransformer

**Определение**: Позволяет применить разные трансформации к разным группам столбцов в одном пайплайне. Например, числовые масштабировать, категориальные – закодировать.

## MAE (Mean Absolute Error)

**Определение**: Средняя абсолютная ошибка регрессии. (1/n) Σ|yᵢ – ŷᵢ|.

## RMSE (Root Mean Squared Error)

**Определение**: Корень из среднего квадрата ошибки; сильнее штрафует за большие отклонения. √[(1/n) Σ(yᵢ – ŷᵢ)²].

## R² (коэффициент детерминации)

**Определение**: Доля дисперсии, объяснённая моделью. 1 – Σ(yᵢ–ŷᵢ)² / Σ(yᵢ–ȳ)².

## Accuracy

**Определение**: Доля правильных ответов в классификации. (TP+TN)/(TP+TN+FP+FN). Бесполезна при сильном дисбалансе классов.

## Precision (точность)

**Определение**: Доля объектов, отнесённых к классу 1, которые действительно являются классом 1. TP/(TP+FP).

## Recall (полнота)

**Определение**: Доля реальных объектов класса 1, правильно предсказанных. TP/(TP+FN).

## F1-score

**Определение**: Среднее гармоническое precision и recall. 2·(Prec·Rec)/(Prec+Rec).

## ROC-AUC (Area Under ROC Curve)

**Определение**: Площадь под ROC-кривой (зависимость True Positive Rate от False Positive Rate). Значение 0,5 – случайное угадывание, 1,0 – идеальный классификатор. sklearn.metrics.roc_auc_score, roc_curve. Широко используется для бинарной классификации, устойчива к дисбалансу классов.

## Дисбаланс классов

**Определение**: Ситуация, когда один класс (например, «клиент не ушёл») значительно преобладает над другим.

## Accuracy парадокс

**Определение**: При дисбалансе accuracy может быть высокой, даже если модель ничего не предсказывает (всегда предсказывает большинство).

## Pipeline (sklearn)

**Определение**: Объект, объединяющий последовательность трансформаций и финальную модель. Упрощает кросс-валидацию и предотвращает утечку данных.

## fit / transform / fit_transform

**Определение**: fit – вычисляет параметры (например, среднее и дисперсию на обучающей выборке); transform – применяет преобразование; fit_transform – комбинация.

## Утечка данных (data leakage)

**Определение**: Использование информации из тестовой выборки при вычислении статистик (например, масштабирование на всём датасете). Недопустимо.

## Порог классификации (threshold)

**Определение**: Значение, выше которого вероятность относит объект к классу 1 (обычно 0.5).

## Влияние порога на метрики

**Определение**: Понижение порога увеличивает recall (находим больше объектов класса 1) за счёт снижения precision (больше ложных срабатываний). Повышение порога – наоборот.

## Нормализация (normalization)

**Определение**: Приведение признаков к единому масштабу.

## Train (обучающая выборка)

**Определение**: Часть данных, используемая для подбора весов и коэффициентов модели. На ней модель «видит» ответы.

## Validation (валидационная выборка)

**Определение**: Часть данных, используемая для настройки гиперпараметров и ранней остановки. Не участвует в обучении, но используется для выбора лучшей конфигурации.

## Test (тестовая выборка)

**Определение**: Часть данных, используемая только для финальной оценки качества модели. Запрещено использовать при настройке модели (во избежание утечки данных).

## Стратификация (stratification)

**Определение**: Разбиение, сохраняющее пропорции целевой переменной (например, классов) в каждой выборке. Параметр stratify в train_test_split.

## KFold (k-блочная кросс-валидация)

**Определение**: Метод повторного разбиения данных на k блоков, где каждый блок по очереди становится валидационным. Полезна для небольших датасетов.

## Data Drift

**Определение**: Изменение статистических свойств данных между выборками (например, между train и test). Может привести к ухудшению модели в эксплуатации.

## Covariate Drift

**Определение**: Изменение распределения признаков (X) при неизменном законе p(y|X). Обнаруживается сравнением X_train и X_test.

## Label Drift

**Определение**: Изменение распределения целевой переменной (y). Характерно для задач с меняющейся частотой классов.

## Concept Drift

**Определение**: Изменение связи между признаками и целевой переменной p(y|X). Сложнее всего обнаружить, требует мониторинга во времени.

## PSI (Population Stability Index)

**Определение**: Метрика изменения распределения числового признака между двумя выборками. PSI = Σ (Actual% – Expected%) * ln(Actual%/Expected%). Значения: <0.1 – стабильно, 0.1–0.2 – небольшие изменения, ≥0.2 – значительный дрейф.

## CSI (Characteristic Stability Index)

**Определение**: PSI для категориальных признаков (аналогичная формула по категориям). Позволяет оценить дрейф категориальных переменных.

## KS-тест (Kolmogorov–Smirnov)

**Определение**: Непараметрический тест равенства распределений двух непрерывных выборок. scipy.stats.ks_2samp(). H₀: распределения одинаковы (дрейфа нет). p-value < 0.05 – дрейф значим.

## t-тест Стьюдента

**Определение**: Проверка равенства средних двух выборок (предполагается нормальность или большой объём). scipy.stats.ttest_ind(). H₀: средние равны.

## Коэффициенты линейной регрессии (важность признаков)

**Определение**: Абсолютные значения весов признаков, показывающие их вклад при условии масштабирования. LinearRegression, Lasso (L1-регуляризация обнуляет неважные признаки).

## Feature Importances (деревья)

**Определение**: Частота использования признака при разбиениях в деревьях; суммарное уменьшение неопределённости. RandomForestRegressor, RandomForestClassifier.

## Feature Importances (CatBoost)

**Определение**: Встроенная оценка важности, основанная на среднем улучшении качества при использовании признака. CatBoostRegressor, CatBoostClassifier.

## Нормализация важности (feature importance)

**Определение**: Приведение значений важности к диапазону [0,1] (делением на максимум) для сравнения между моделями. importance / max(importance).

## Доменный признак (feature engineering)

**Определение**: Создан на основе бизнес-логики или физического смысла (например, ИМТ, возрастная группа).

## Математическое преобразование (признака)

**Определение**: Логарифм, экспонента, квадрат, произведение двух признаков для учёта нелинейностей.

## Пороговый (бинарный) признак

**Определение**: Метка, достигает ли числовой признак определённого уровня (например, age > 60).

## Категоризация непрерывного признака

**Определение**: Преобразование в группы (биннинг) с помощью pd.cut.

## Корреляция Пирсона (для отбора признаков)

**Определение**: Линейная связь между числовыми признаками и целевой переменной.

## Корреляция Спирмена (для отбора признаков)

**Определение**: Монотонная связь (ранговая) – менее чувствительна к выбросам.

## t-тест для бинарного признака

**Определение**: Сравнение средних целевой переменной для двух категорий (например, sex).

## ANOVA (F-тест)

**Определение**: Сравнение средних целевой переменной для нескольких категорий (например, region).

## WineQT (датасет)

**Определение**: Датасет с физико-химическими свойствами вина и оценкой качества (целевая переменная quality – целые от 0 до 10).

## Многоклассовая классификация

**Определение**: Предсказание одной из нескольких дискретных меток (качество вина).

## Процесс подбора признаков (Feature Selection)

**Определение**: 1. Статистический анализ (корреляции, t-test, ANOVA). 2. Создание новых признаков. 3. Оценка важности через модели. 4. Итеративное улучшение.

## Проклятие размерности (curse of dimensionality)

**Определение**: Рост числа признаков приводит к заметному увеличению объёма данных, необходимого для статистически значимых выводов; расстояния между точками теряют различимость. Проявляется при D ~ n или D > n, где D – число признаков, n – число наблюдений.

## Линейное понижение размерности

**Определение**: Метод, который проецирует данные на линейное подпространство меньшей размерности. Пример: PCA.

## Нелинейное понижение размерности

**Определение**: Метод, который ищет нелинейный способ понизить размерность с сохранением каких-то характеристик, зачастую расстояния. Примеры: t-SNE, UMAP.

## PCA (Principal Component Analysis)

**Определение**: Линейный метод понижения размерности, находящий ортогональные направления (главные компоненты), вдоль которых дисперсия данных максимальна. sklearn.decomposition.PCA.

## Главная компонента (principal component)

**Определение**: Нормализованный собственный вектор ковариационной матрицы, соответствующий наибольшему собственному значению. Компоненты упорядочены по убыванию объяснённой дисперсии.

## Объяснённая дисперсия (explained variance)

**Определение**: Доля дисперсии исходных данных, сохраняемая при проекции на данную главную компоненту. Доступна в pca.explained_variance_ratio_.

## Суммарная объяснённая дисперсия

**Определение**: Сумма долей дисперсии для первых k компонент. Показывает, сколько информации потеряно при сжатии до k измерений. Используется для выбора оптимальной размерности (правило локтя).

## Правило локтя (elbow rule) для PCA

**Определение**: Выбор количества компонент, после которого прирост суммарной объяснённой дисперсии становится незначительным. Визуализируется графиком explained_variance_ratio_ в порядке убывания.

## Матрица компонент pca.components_

**Определение**: Матрица размера (n_components, n_features), строки которой являются главными компонентами (направлениями в исходном пространстве). Каждая компонента может быть визуализирована как «изображение» (например, для MNIST).

## Стандартизация для PCA

**Определение**: Данные должны быть центрированы (вычтено среднее) и часто масштабированы (приведены к единичной дисперсии). Влияет на результат, поскольку PCA чувствителен к масштабу признаков.

## t-SNE (t-distributed Stochastic Neighbor Embedding)

**Определение**: Нелинейный метод понижения размерности, основанный на вероятностном сходстве точек и использующий распределение Стьюдента для низкоразмерного отображения. sklearn.manifold.TSNE (также openTSNE, поддерживает размерность 2).

## Перплексия (perplexity) в t-SNE

**Определение**: Эффективное число ближайших соседей. Типичные значения: 5–50. Меньшие значения подчёркивают локальную структуру, большие – глобальную.

## Скорость обучения (learning rate) в t-SNE

**Определение**: Шаг градиентного спуска при оптимизации расхождения Кульбака–Лейблера. Типичные значения: 10–1000. При слишком малой – медленная сходимость, при слишком большой – расходимость.

## Инициализация (init) в t-SNE

**Определение**: Начальное расположение точек в низкоразмерном пространстве. 'random' – случайное, 'pca' – с помощью PCA (часто даёт лучшие результаты). Параметр init в TSNE.

## Расхождение Кульбака–Лейблера (KL divergence) в t-SNE

**Определение**: Мера различия между распределением вероятностей соседства в исходном и низкоразмерном пространстве. t-SNE минимизирует KL(P||Q). KL(P||Q) = Σ_{i≠j} p_{ij} log(p_{ij}/q_{ij}).

## Интерпретация t-SNE

**Определение**: Расстояния между кластерами не имеют количественного смысла, важна только относительная близость точек внутри кластеров. t-SNE не сохраняет глобальные расстояния и масштабы.

## UMAP (Uniform Manifold Approximation and Projection)

**Определение**: Нелинейный метод понижения размерности, улучшение t-SNE. Быстрее t-SNE и лучше сохраняет глобальную структуру. Часто даёт кластеры, похожие на t‑SNE, но имеет тенденцию сжимать их. Библиотека umap-learn.

## n_neighbors в UMAP

**Определение**: Аналог перплексии; размер локального окружения. Меньшие значения фокусируются на локальной структуре, большие – на глобальной. Параметр n_neighbors в umap.UMAP.

## min_dist в UMAP

**Определение**: Минимальное допустимое расстояние между точками в низкоразмерном пространстве. Меньшие значения приводят к более плотным кластерам. Параметр min_dist (по умолчанию 0.1).

## Метрика (metric) в UMAP

**Определение**: Функция расстояния, используемая для построения графа соседства. По умолчанию 'euclidean'. Параметр metric.

## Предварительное понижение размерности (перед t-SNE/UMAP)

**Определение**: Применение PCA (например, до 50 компонент) перед t-SNE или UMAP для ускорения работы и шумоподавления. Используется, когда исходная размерность слишком велика.

## Визуализация проекции (2D/3D)

**Определение**: Точечная диаграмма (scatter plot) низкоразмерного представления с цветовой кодировкой по меткам классов. plt.scatter(x, y, c=labels, cmap=...).

## Оценка разделимости кластеров

**Определение**: Насколько хорошо кластеры, соответствующие разным меткам, визуально разделяются в проекции. Не является точной метрикой, но даёт интуитивное представление о структуре данных.

## Визуализация one‑hot признаков (после понижения размерности)

**Определение**: После создания бинарных признаков на основе токенов (число признаков может быть велико) применяют PCA, t-SNE или UMAP для снижения размерности и визуализации разделимости классов. Позволяет оценить, насколько признаки информативны для классификации.

## ROC-кривая (Receiver Operating Characteristic)

**Определение**: График зависимости True Positive Rate (TPR) от False Positive Rate (FPR) при варьировании порога классификации. sklearn.metrics.roc_curve.

## AUC (Area Under Curve)

**Определение**: Площадь под ROC-кривой. Чем ближе к 1, тем лучше модель разделяет классы. Значение 0,5 соответствует случайному угадыванию. sklearn.metrics.roc_auc_score.

## Частота токена (token frequency)

**Определение**: Количество вхождений токена в текстах определённого класса или во всём корпусе. Вычисляется с помощью collections.Counter.

## Разница частот (frequency difference) токенов

**Определение**: Абсолютная разность долей встречаемости токена в двух классах (например, позитивном и негативном). Позволяет отобрать наиболее дискриминативные токены для построения признаков.

## One‑hot кодирование токенов (для классификации текстов)

**Определение**: Для каждого отобранного токена создаётся признак: 1, если токен встретился в тексте, 0 – в противном случае.

## Логистическая регрессия на one‑hot токенах

**Определение**: Простая, но часто эффективная модель для бинарной классификации текстов (например, определение тональности).

## Разделение выборки (текстовая классификация)

**Определение**: Данные делятся на обучающую (train) и тестовую (test) с сохранением пропорции классов (стратификация).

## Обучение модели (текстовая классификация)

**Определение**: Модель (например, LogisticRegression) обучается на train‑наборе.

## Прогноз (текстовая классификация)

**Определение**: Для test‑набора вычисляются вероятности принадлежности к классу 1.

## Поиск похожих предложений (semantic similarity search)

**Определение**: Задача нахождения в корпусе предложений, наиболее близких по смыслу к заданному запросу. Используется косинусное сходство между векторами предложений (полученными суммированием эмбеддингов токенов). Полезна для анализа текстовых данных, рекомендаций, кластеризации.

## Геоданные (spatial data)

**Определение**: Данные, содержащие информацию о географических координатах (широта, долгота) или геометрических объектах (точки, линии, полигоны). Используются для картографического анализа, пространственных соединений.

## GeoDataFrame (geopandas)

**Определение**: Таблица, расширяющая pd.DataFrame специальным столбцом geometry, содержащим геометрические объекты (точки, полигоны и т.д.). gpd.GeoDataFrame(). Позволяет выполнять пространственные операции.

## CRS (Coordinate Reference System)

**Определение**: Система координат, задающая, как географические координаты (широта, долгота) проецируются на плоскость. EPSG:4326 – WGS84 (географические координаты, градусы). to_crs() – перепроецирование.

## Точечная геометрия (Point geometry)

**Определение**: Объект shapely.geometry.Point, представляющий одну географическую точку (например, место посадки/высадки). Создаётся через gpd.points_from_xy(x, y, crs=...).

## Полигональная геометрия (Polygon / MultiPolygon)

**Определение**: Объект, задающий границы территориальной единицы (район, округ, страна) в виде многоугольника (или набора многоугольников). Загружается из GeoJSON-файлов.

## Пространственное соединение (spatial join)

**Определение**: Операция, сопоставляющая каждому объекту (точке) атрибуты полигона, в который эта точка попадает (предикат within). gpd.sjoin(gdf_points, gdf_polygons, how='left', predicate='within').

## Предикат within (пространственный)

**Определение**: Проверяет, содержится ли геометрия левого операнда полностью внутри геометрии правого. Используется для определения принадлежности точки к полигону.

## Plotly Express (px)

**Определение**: Высокоуровневый интерфейс для быстрого создания интерактивных графиков (включая карты). plotly.express.scatter_map – точечная карта.

## Plotly Graph Objects (go)

**Определение**: Низкоуровневый интерфейс для детальной настройки сложных графиков, включая subplots и связанные фильтры. make_subplots(), go.Scattermap(), go.Scatter().

## Scattermap (карта-рассеяние в Plotly)

**Определение**: Слой точечных данных, наложенный на карту (обычно OpenStreetMap или стиль Mapbox). go.Scattermap(lat=..., lon=..., mode='markers', marker=..., text=...).

## Субплоты (subplots) в Plotly

**Определение**: Группа из нескольких графиков, расположенных в сетке и связанных общими данными/фильтрами. fig = make_subplots(rows, cols, subplot_titles), затем fig.add_trace(trace, row, col).

## Настраиваемое всплывающее окно (customdata + hovertemplate) в Plotly

**Определение**: Механизм передачи дополнительных данных для отображения при наведении курсора на точку. customdata – массив данных; hovertemplate – строка с плейсхолдерами %{customdata[...]}.

## Фильтрация через виджеты (updatemenus / buttons) в Plotly

**Определение**: Элементы управления, позволяющие динамически показывать/скрывать слои на графике. updatemenus с buttons; visible массива трасс управляется через args.

## Связанные графики (cross‑filtering) в Plotly

**Определение**: Выбор категории в одном графике автоматически отображает соответствующие подмножества точек на других графиках. Реализуется через updatemenus и изменение visible всех трасс.

## Экспорт в HTML (Plotly)

**Определение**: Сохранение интерактивного графика в отдельный HTML-файл для просмотра в браузере. fig.write_html('filename.html').

## Геодезическое расстояние (geodesic distance)

**Определение**: Расстояние между двумя точками на поверхности земного эллипсоида (в отличие от евклидова). geopy.distance.geodesic(pointA, pointB, ellipsoid='GRS-80').miles (или километры).

## Вычисление distance_between_points

**Определение**: Новый признак – прямое расстояние между двумя точками (например, между координатами посадки и высадки). Используется для сравнения с фактическим пройденным расстоянием (например, из одометра).

## Аномалии расстояния (геоаналитика)

**Определение**: Случаи, когда фактическое расстояние значительно превышает геодезическое – возможный выброс или ошибка данных. Отфильтровываются или анализируются отдельно.

## Час события (pickup_hour)

**Определение**: Целочисленный признак (0–23), извлечённый из временной метки. Позволяет анализировать суточные паттерны.

## Стоимость мили (cost_per_unit)

**Определение**: Отношение стоимости услуги к пройденному расстоянию (например, fare_amount / trip_distance). Характеризует тарификацию, может выявлять аномалии.

## Фильтрация по ограничивающему прямоугольнику (bounding box)

**Определение**: Ограничение координат диапазонами широт и долгот, соответствующими исследуемой территории. Удаляет выбросы (некорректные GPS-координаты).
